图片资源请求如下：

设置了display:none属性的元素，图片不会渲染出来，但会加载。
原理：设置了display:none的元素，还是会构建DOM树的,只不过在渲染的时候，不会计算位置而已.
当父级元素设置了display:none之后，他的子集的背景就不会渲染也不会参与构建，所以图片存在于display:none子集里面的背景图片是不会加载也不会渲染的，而<img /<是会参与构建DOM树的，所以也会加载
